# Native Config Overlay: Provider Full-Access Runtime Harness

[Back to native-config-overlay](../native-config-overlay.md)

Riido AI Agent executes provider CLIs on the user's PC. If providers do not
receive enough runtime permission, the product cannot perform user-requested work.
C4's canonical direction is therefore provider-native full-access/trusted
runtime envelopes selected explicitly by adapters and governed by the
C7/C4/C5/C6 harness.

Codex canonical launch shape:

```txt
codex --sandbox danger-full-access app-server --listen stdio://
```

This is the only Codex sandbox selection generated by daemon. Caller
`CustomArgs`, client codegen, SaaS assignment payload, and policy bundle cannot
change it. C4 drops caller-provided `--sandbox`, `--sandbox=*`, `-s`, `-s=*`,
`-c`, `--config`, `--enable`, `--disable`, `--yolo`, and
`--dangerously-bypass-approvals-and-sandbox`, then records `DroppedArgs`.

Harness responsibilities:

- immutable assignment snapshot and prompt/native-config placement
- daemon-selected workdir and result evidence root
- provider process start/stop/cancel and terminal result
- provider log/progress redaction
- 5-second heartbeat and 20-second stale orphan release
- runtime slot, lease, fencing token, and task-level active assignment policy
- provider real-integration gate and workdir side-effect verification

Redaction marker/catalog details remain owned by
[`../security-redaction.md`](../security-redaction.md).

This replaces the RIID-4881 permission-profile experiment. The new model admits
full-access execution explicitly and manages its risk through harness, lease,
heartbeat, and evidence.

Claude, Cursor, and OpenClaw follow the same meta-model, but provider-specific
trust flags need their own SSOT, command builder, and integration evidence PRs.
Until promoted, Claude `bypassPermissions`, Cursor `--yolo`, and similar
provider-defined approval bypass surfaces remain unsafe bypass surfaces.

C7's key question is not whether full-access is default. It is which trusted
runtime envelope was adopted for the provider and what harness evidence governs
that envelope. Provider adoption state is owned by
[`provider-runtime.md`](../../provider-runtime.md).
